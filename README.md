# flyrank-capstone
This is my frontend capstone project.

## Critique

Your README is clear but very minimal — essentially a title and one sentence. For a capstone repo, that can read as unfinished rather than intentional.

**What works**
- The `# flyrank-capstone` heading matches the repo name.
- You state upfront that this is a frontend capstone project.

**What’s missing**
- **Purpose** — What is FlyRank? What problem does it solve?
- **Tech stack** — Your project uses React/Next.js (per `CLAUDE.md`), but the README doesn’t mention that.
- **Getting started** — No install or run steps (even a “coming soon” note helps).
- **Structure** — Professional READMEs usually use sections (`About`, `Tech Stack`, `Getting Started`) so readers know where to look.

That’s fine for day one, but reviewers, instructors, or employers opening the repo won’t learn much beyond “this exists.”

---

## One improvement

**Add a short “About” section with a one-line tagline and your tech stack.**

That turns a placeholder into something that looks deliberate, even before the app is built. For example:

```markdown
# flyrank-capstone

A frontend capstone project built with React and Next.js.

## About

FlyRank is [brief description of what the app does — e.g., "a tool that helps users track and compare flight prices"].

## Tech Stack

- React
- Next.js
- JavaScript
```

Replace the bracketed line with your actual project goal once you know it. Even a rough sentence (“a web app for …”) is enough for now.
